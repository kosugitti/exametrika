# CLAUDE.md - Development Guide for exametrika

## Package Overview

`exametrika` is an R package for Test Data Engineering based on Shojima
(2022, <ISBN:978-9811699856>). It provides psychometric analysis tools:
CTT, IRT, GRM, LCA, LRA, Biclustering, BNM, LDLRA, LDB, BINET.

- **Current version**: 1.10.0 (development)
- **CRAN version**: 1.8.1
- **GitHub Release**: v1.9.0
- **License**: MIT
- **Website**: <https://kosugitti.github.io/exametrika/>
- **Language**: English (Japanese guide available as
  `vignettes/guide-ja.Rmd`)

## Build & Test Commands

``` bash
# Run all tests (requires devtools for Rcpp compilation)
Rscript -e 'devtools::test()'

# Run a single test file
Rscript -e 'devtools::load_all(); testthat::test_file("tests/testthat/test-irt.R")'

# R CMD check (full package check)
R CMD build . && R CMD check --as-cran exametrika_*.tar.gz

# Generate documentation from roxygen comments
Rscript -e 'roxygen2::roxygenise()'
```

## Coding Conventions

- **Naming**: Public functions use PascalCase (e.g., `ItemStatistics`,
  `GridSearch`). Return value fields use snake_case (e.g., `n_class`,
  `log_lik`). Old PascalCase fields (`Nclass`, `LogLik`) are deprecated
  but retained for backward compatibility until v2.0.0.
- **S3 dispatch**:
  [`Biclustering()`](https://kosugitti.github.io/exametrika/reference/Biclustering.md),
  [`LRA()`](https://kosugitti.github.io/exametrika/reference/LRA.md),
  and
  [`Biclustering_IRM()`](https://kosugitti.github.io/exametrika/reference/Biclustering_IRM.md)
  dispatch on data type (binary/ordinal/rated/nominal) via
  [`UseMethod()`](https://rdrr.io/r/base/UseMethod.html). Plot and print
  methods dispatch on `exametrika` class.
- **Model fit**: All models return `TestFitIndices` as a 16-field
  `ModelFit` object via
  [`calcFitIndices()`](https://kosugitti.github.io/exametrika/reference/calcFitIndices.md)
  in `00_ModelFitModule.R`.
- **Plotting**:
  [`plot.exametrika()`](https://kosugitti.github.io/exametrika/reference/plot.exametrika.md)
  in `00_exametrikaPlot.R` dispatches to model-family plot files
  (`00_plot_*.R`).
- **Progress output**: Use
  [`message()`](https://rdrr.io/r/base/message.html) (not
  [`cat()`](https://rdrr.io/r/base/cat.html)/[`print()`](https://rdrr.io/r/base/print.html))
  for progress messages, controlled by `verbose` parameter.
- **Rcpp**: Performance-critical code in `src/` (GRM gradient,
  polychoric correlation).
- **apply() caution**: Never use `apply(X, 2, FUN)` when the result is
  used as a list. Use
  `lapply(seq_len(ncol(X)), function(j) FUN(X[, j]))` instead, because
  [`apply()`](https://rdrr.io/r/base/apply.html) returns a matrix (not a
  list) when all columns produce same-length results.

## Testing

- **Framework**: testthat (edition 3)
- **Reference data**: Mathematica implementation outputs in
  `tests/testthat/fixtures/mathematica_reference/` (85 CSV files) used
  for cross-validation
- **Test naming**: `test-<model>.R` for model tests (e.g., `test-irt.R`,
  `test-lca.R`)
- **Test dependencies**: Only `testthat` (no tidyverse/readxl — migrated
  to CSV/base-R in v1.10.0)
- **CI/CD**: GitHub Actions — `R-CMD-check.yaml` (multi-platform),
  `test-coverage.yaml` (coverage report), `pkgdown.yaml` (site
  deployment to gh-pages), `rhub.yaml` (R-hub v2)
- **Build config**: `.Rbuildignore` excludes `develop/`, `docs/`,
  `.claude/`, `CLAUDE.md`, `tools/`, `.github/`, etc. Tests and inst are
  NOT excluded (fixed in v1.10.0)
- **Current status**: 873 tests, all passing (FAIL 0 \| WARN 0 \| SKIP 0
  \| PASS 873)
- **R CMD check**: 0 errors \| 0 warnings \| 1 note (“unable to verify
  current time” — transient network issue)

## Known Technical Debt

### TODO Items (`tests/testthat/test-lra-ordinal.R`)

- **L149**: Investigate `nobs` handling unification — R uses per-item
  nobs (`colSums(Z)`) while Mathematica uses total nobs. Affects RMSEA,
  CAIC, BIC for datasets with significant missing data (e.g., J15S3810).
- **L171**: Related test-level nobs handling for ordinal LRA.

### Deprecated Functions (to be removed in v2.0.0)

- [`IRM()`](https://kosugitti.github.io/exametrika/reference/IRM.md) →
  use
  [`Biclustering_IRM()`](https://kosugitti.github.io/exametrika/reference/Biclustering_IRM.md)
- [`StrLearningGA_BNM()`](https://kosugitti.github.io/exametrika/reference/StrLearningGA_BNM.md)
  → use
  [`BNM_GA()`](https://kosugitti.github.io/exametrika/reference/BNM_GA.md)
- [`StrLearningPBIL_BNM()`](https://kosugitti.github.io/exametrika/reference/StrLearningPBIL_BNM.md)
  → use
  [`BNM_PBIL()`](https://kosugitti.github.io/exametrika/reference/BNM_PBIL.md)

  collapsed Gibbs sampler

  ordinal EM with cumulative normalization
- **Shared Gibbs core** (`R/00_IRM_Gibbs_CORE.R`): Extracted collapsed
  Gibbs sampler + helpers
- **Biclustering_IRM S3 generic**: dispatches to
  .binary/.nominal/.ordinal by data type

- CI: removed Codecov upload from test-coverage workflow
- TestStatistics docs: stem-and-leaf plot example
- Test suite modernization (Excel→CSV, tidyverse dependency removal)
- pkgdown migration, CI/CD, .Rbuildignore cleanup
