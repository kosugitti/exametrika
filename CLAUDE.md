# CLAUDE.md - Development Guide for exametrika

## Package Overview

`exametrika` is an R package for Test Data Engineering based on Shojima (2022, ISBN:978-9811699856).
It provides psychometric analysis tools: CTT, IRT, GRM, LCA, LRA, Biclustering, BNM, LDLRA, LDB, BINET.

- **Current version**: 1.9.0 (development complete, CRAN submission pending)
- **CRAN version**: 1.8.1
- **License**: MIT
- **Website**: https://kosugitti.github.io/exametrika/
- **Language**: English (Japanese guide available as `vignettes/guide-ja.Rmd`)

## Build & Test Commands

# Run all tests (requires devtools for Rcpp compilation)

## Coding Conventions

- **Naming**: Public functions use PascalCase (e.g., `ItemStatistics`, `GridSearch`).
  Return value fields use snake_case (e.g., `n_class`, `log_lik`). Old PascalCase
  fields (`Nclass`, `LogLik`) are deprecated but retained for backward compatibility
  until v2.0.0.
- **S3 dispatch**: `Biclustering()` and `LRA()` dispatch on data type
  (binary/ordinal/rated/nominal) via `UseMethod()`. Plot and print methods
  dispatch on `exametrika` class.
- **Model fit**: All models return `TestFitIndices` as a 16-field `ModelFit` object
  via `calcFitIndices()` in `00_ModelFitModule.R`.
- **Plotting**: `plot.exametrika()` in `00_exametrikaPlot.R` dispatches to
  model-family plot files (`00_plot_*.R`).
- **Progress output**: Use `message()` (not `cat()`/`print()`) for progress messages,
  controlled by `verbose` parameter.
- **Rcpp**: Performance-critical code in `src/` (GRM gradient, polychoric correlation).
- **apply() caution**: Never use `apply(X, 2, FUN)` when the result is used as a list.
  Use `lapply(seq_len(ncol(X)), function(j) FUN(X[, j]))` instead, because `apply()`
  returns a matrix (not a list) when all columns produce same-length results.

## Testing

- **Framework**: testthat (edition 3)
- **Reference data**: Mathematica implementation outputs in `tests/testthat/fixtures/mathematica_reference/`
  (85 CSV files) used for cross-validation
- **Test naming**: `test-<model>.R` for model tests (e.g., `test-irt.R`, `test-lca.R`)
- **Test dependencies**: Only `testthat` (no tidyverse/readxl ŌĆö migrated to CSV/base-R in v1.9.0)

  `tools/`, `.github/`, etc. Tests and inst are NOT excluded (fixed in v1.9.0)
